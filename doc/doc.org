Назначение
Система представляет возможность проверить ваш код на соответствие кода принятым на предприятии стандартам (например требованиям описаным на ИТС)
Так же возможно добавление собственных правил и требований

В идеале эта система будет уметь то же что и StyleCop + Doxygen + Ndepend 
Т.е. возможность анализа текста, анализа метаданных анализ AST, расчет метрик 


Типы правил

*token* -  Принимает следующие параметры - 
1. text - строка текст токена
2. type - символ  тип токена
3. start - codepoint начало токена
4. end -  codepoint  конец токена
5. file - file-entry обрабатываемый файл

Может использоваться для анализа текста и положения лексемы, но не несет информации о смысле лексекмы (например анализируя идентификатор нельзя выяснить что это - переменная или имя функции), так же нельзя получить информацию о предыдущих последующих токенах. Пример использования - файл ../rule/check-keyword.rule


*lexer* -  Принимает следующие параметры - 
1. text - строка текст токена
2. type - символ  тип токена
3. start - codepoint начало токена
4. end -  codepoint  конец токена
5. index - номер в списке
6. stream - список лексем
5. file - file-entry обрабатываемый файл

Правило может выяснить где находиться лексема, и исходя из этого уже анализировать все что хочется
например - отсутвие комментария перд определением функции
